Kubernetes（k8s）的Service Ingress详细介绍

在版本，是层概念。在版本，新增了版，用来表示层服务资料信息可实现层代理地址：官方网站：部署下载文件运行配置文件给权限，不然会出现问题。我遇见这个问题，链接，你们可以参考一下下载暴露端口配置文件代理访问。。。。。。。

Feb 06, 2020
阅读时长: 3 分钟

**在 Kubernetes v1.0 版本， Service 是 “4层”（TCP/UDP over IP）概念。 在 Kubernetes v1.1 版本，新增了 Ingress API（beta 版），用来表示 “7层”（HTTP）服务 **

资料信息 【可实现7层代理】
Ingress-Nginx github 地址：https://github.com/kubernetes/ingress-nginx

Ingress-Nginx 官方网站：https://kubernetes.github.io/ingress-nginx/

部署 Ingress-Nginx 

下载Ingress-Nginx 文件

1

wget https://raw.githubusercontent.com/kubernetes/ingress-nginx/nginx-0.28.0/deploy/static/mandatory.yaml

运行配置文件

1

​​​​chmod -R 777 /var/lib/docker

给docker权限，不然会出现问题。

我遇见这个问题，链接，你们可以参考一下

kubectl get pod -n ingress-nginx

1

 kubectl apply -f service-nodeport.yaml

Nginx 进行重写 

名称描述值
nginx.ingress.kubernetes.io/rewritetarget必须重定向流量的目标URI 串
nginx.ingress.kubernetes.io/sslredirect
 		指示位置部分是否仅可访问SSL（当Ingress包含证书时 默认为True）
 		布尔
nginx.ingress.kubernetes.io/forcessl-redirect即使Ingress未启用TLS，也强制重定向到HTTPS
 		布尔
nginx.ingress.kubernetes.io/approot定义Controller必须重定向的应用程序根，如果它在'/'上 下文中
 		串
nginx.ingress.kubernetes.io/useregex指示Ingress上定义的路径是否使用正则表达式
 		布尔
